# AGENTS.md — Multipage Contractor Website

## Project

Multipage marketing website for **Home Improvement Contractors** serving the US market — roofers, landscapers, electricians, plumbers, HVAC, painters, masons, remodelers, and general contractors. Built with Astro, content-driven via structured JSON files. SEO-optimized, English-only, targeting local service areas.

> **Current client**: Masonry & Hardscape contractor (company name TBD). The template is built generic-first; this client is the first instantiation.

### Dual-Repo Strategy

This project produces **two repositories**:

| Repo | Name | Purpose |
|------|------|---------|
| **Template** | `website-multipages` | Reusable foundation for any home improvement contractor. Contains all framework code: components, layouts, utilities, design system, content schemas. Client-agnostic. |
| **Client** | `[company-name]` | Production website for the current client (Masonry & Hardscape). Sources template as a base, layers client-specific JSON content on top. |

**Separation principle**: Client-specific data lives STRICTLY in JSON content files (`CONTENT.json`, `LANDINGS.json`, `BLOGS.json`). All `.astro` components, layouts, and utilities remain generic and reusable. The JSON layer IS the customization point.

**Consequence for development**:
- Every component must accept data via props from JSON, never hardcode client info
- Template repo must work out-of-the-box with any contractor's JSON data
- Client repo can clone or fork the template, then fill in their JSON files
- No client data in commits to the template repo — use sample/placeholder data only

## Tech Stack

- **Framework**: Astro (static generation, `.astro` components)
- **Styling**: Tailwind CSS (utility-first)
- **Design Intelligence**: `ui-ux-pro-max` skill — provides industry-specific design system generation (Home Services category), 67 UI styles, 161 color palettes, typography pairings, and pre-delivery UX checklist
- **Content**: JSON-driven (`CONTENT.json`, `LANDINGS.json`, `BLOGS.json`)
- **Fonts**: Google Fonts (to be selected by ui-ux-pro-max design system)
- **Icons**: SVG via Heroicons or Lucide (no emojis as icons)
- **Language**: TypeScript for scripts, Astro for templates
- **Package Manager**: npm
- **Runtime**: Node.js (LTS)

## Content Workflow

The project uses a **JSON-driven content pipeline** with clear separation of concerns:

1. **CLIENT DATA fields** — filled manually by the client/project owner (names, phone, address, service names, social URLs, brand colors)
2. **AI COPY fields** — marked as `[PLACEHOLDER]` in JSON; the agent writes copy based on client data when invoked
3. **Schema contracts**: each JSON file has `_instructions` defining word counts and constraints per field

### Content files

| File | Purpose | Pages fed |
|------|---------|-----------|
| `CONTENT.json` | Global company info, services, mission/vision, social links, brand colors, phrases | Home, About, Services, Footer, global SEO |
| `LANDINGS.json` | Per-service landing pages with hero + 3 content sections each | `/landing/[slug]` |
| `BLOGS.json` | Blog posts with full HTML body, meta tags, excerpts | `/blog/[slug]` |

### Agent instructions for content generation

- **Tone**: Professional, trustworthy, authoritative — match the construction/contractor industry
- **Market**: US homeowners, English, local SEO focus
- **Copy rules**: headlines ≤12 words, paragraphs 50–60 words, mission/vision exactly 50 words, blog 600–1500 words, titles ≤60 chars, meta descriptions ≤160 chars
- **SEO**: Include city/area names naturally, use semantic HTML, proper heading hierarchy
- **Never**: AI-sounding fluff, emojis in icons, generic stock-photo descriptions

## Conventions

### Code
- **Components**: PascalCase (`HeroSection.astro`, `ServiceCard.astro`)
- **Layouts**: PascalCase, suffixed with `Layout` (`BaseLayout.astro`, `LandingLayout.astro`)
- **Pages**: kebab-case in URLs (`/landing/kitchen-remodeling`)
- **Images**: kebab-case, descriptive (`kitchen-hero.webp`, `blog-paver-installation.webp`)
- **No AI attribution**: never add "Co-Authored-By" or AI credits to commits
- **Conventional commits**: `feat:`, `fix:`, `content:`, `style:`, `seo:`
- **No build on commit**: never run build automatically; only when explicitly asked

### Accessibility (mandatory)
- Light mode only (construction industry standard)
- Text contrast 4.5:1 minimum
- Focus states visible for keyboard navigation
- `prefers-reduced-motion` respected
- All interactive elements have `cursor-pointer`
- Hover states with smooth transitions (150–300ms)
- Responsive breakpoints: 375px, 768px, 1024px, 1440px

### Performance
- Images in WebP format with responsive `srcset`
- Font optimization (subset, swap, preload critical)
- Zero JavaScript by default (Astro static); add islands only when needed
- LCP target < 2.5s, CLS < 0.1
